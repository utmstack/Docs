# Migrating from IBM QRadar to UTMStack

## Introduction

This guide provides instructions for migrating from IBM QRadar to UTMStack, an open-source security platform designed for Ubuntu 22.04 LTS. UTMStack offers advanced threat detection, log management, and compliance features, making it a robust alternative to traditional SIEM solutions.

## Overview of UTMStack

UTMStack is a comprehensive cybersecurity platform that integrates SIEM and XDR capabilities. It is designed to provide real-time threat detection and response, leveraging a proprietary data visualization and analysis engine. UTMStack supports a wide range of integrations and offers customizable dashboards and reports without the need for coding.

### Key Features

- **Log Management and Correlation**: Efficiently manage and correlate logs for enhanced security insights.
- **Threat Detection and Response**: Identify and respond to threats in real-time.
- **Compliance Management**: Generate reports for compliance with standards like HIPAA, GLBA, GDPR, and SOC.
- **Integration Support**: Compatible with platforms such as Azure, AWS, and various SaaS applications.

## System Requirements

Before migrating to UTMStack, ensure your system meets the following requirements:

- **Operating System**: Ubuntu 22.04 LTS
- **CPU**: 64-bit x86-64 v2 architecture
- **Resources**:
  - Minimum: 4 CPU cores, 16GB RAM, 100GB disk space
  - Additional space required for each integration: 1GB

For detailed system requirements, refer to the [UTMStack System Requirements](https://documentation.utmstack.com/v10/Installation/system_requirements).

## Installation Guide

### Preparing for Installation

1. **Update Packages List**:
   ```bash
sudo apt update
   ```
2. **Install WGET**:
   ```bash
sudo apt install wget
   ```
3. **Download Installer**:
   ```bash
wget http://github.com/utmstack/UTMStack/releases/latest/download/installer
   ```
4. **Change to Root User**:
   ```bash
sudo su
   ```
5. **Set Execution Permissions**:
   ```bash
chmod +x installer
   ```

### Running Installation

Execute the installer without parameters to begin the installation process:

```bash
./installer
```

After installation, access the UTMStack login page using HTTPS with the server name or IP address. Use the default credentials found in `/root/utmstack.yml`.

### Required Ports

- **22/TCP**: Secure Shell (SSH)
- **80/TCP**: UTMStack Web-based GUI Redirector
- **443/TCP**: UTMStack Web-based GUI
- **9090/TCP**: Cockpit Web-based GUI for Servers

For security, it is recommended to create firewall rules to restrict access to these ports.

## Configuration and Customization

UTMStack offers flexible configuration options, including:

For more information on configuration options, visit the [UTMStack Configuration Guide](https://documentation.utmstack.com/v10/configuration/application_settings).

Migrating from IBM QRadar to UTMStack involves preparing your system according to the specified requirements, installing UTMStack using the provided installer, and configuring the platform to suit your organization's needs.
