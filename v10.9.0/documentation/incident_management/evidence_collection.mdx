# Evidence Collection

## Introduction

In the realm of cybersecurity, collecting and preserving digital evidence is a critical component of incident management. This guide provides an overview of the evidence collection process within the UTMStack platform, a comprehensive cybersecurity solution designed for hybrid environments. UTMStack integrates various security services, including SIEM, threat detection, incident response, and compliance management, to streamline the management and investigation of security incidents.

## UTMStack Overview

UTMStack is an open-source Unified Threat Management platform that offers a wide range of security services. It is designed to be cost-effective and user-friendly, providing a single interface for managing multiple cybersecurity functions. Key features include:

- **Log Management (SIEM):** Real-time log collection and correlation, enabling forensic analysis and compliance reporting.
- **Threat Detection and Response:** Utilizes rule-based, heuristic, and machine learning engines to identify and respond to threats.
- **Compliance Management:** Supports compliance with regulations like HIPAA, GLBA, SOC 2, and more.
- **Incident Response:** Automated and on-demand incident response capabilities, including host lockdown and IP blocking.
- **Threat Intelligence:** Correlates data from various threat intelligence feeds to enhance detection capabilities.

## Incident Management and Evidence Collection

### Incident Tracking

UTMStack provides a robust framework for tracking incident history, which is essential for evidence collection. The schema for tracking incidents includes:

- **Incident History:** Captures details such as action type, action date, and the user who created the action.
- **Action Types:** Includes actions like INCIDENT_CREATED, INCIDENT_MERGED, INCIDENT_COMPLETED, and more.

### Data Schemas

The platform defines several JSON schema objects used for log analysis and incident management. These schemas specify required fields and types for entities, ensuring that all relevant data is captured and preserved. Key schemas include:

- **LogAnalyzerQuery:** Used for querying logs, with properties like name, description, and creation date.
- **PolicySettings:** Manages data retention policies, specifying how long data should be stored.

## Dashboard and Reporting

UTMStack's dashboard provides a centralized interface for monitoring and analyzing security data. Users can create and customize dashboards to visualize log data, alerts, and network traffic. This feature enhances the ability to perform forensic analysis and generate compliance reports.

### Creating Dashboards

1. Navigate to the **Dashboard** tab after logging into UTMStack.
2. Click on **New Dashboard** to create custom visualizations from log, event, and alarm data.

## Summary

UTMStack is a powerful tool for managing and investigating security incidents, offering integrated services that simplify the process of evidence collection. By leveraging its comprehensive set of features, organizations can effectively track incidents, manage data retention, and perform detailed forensic analysis. The platform's user-friendly interface and customizable dashboards further enhance its utility in maintaining robust cybersecurity defenses.
