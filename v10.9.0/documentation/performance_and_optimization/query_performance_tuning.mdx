# Query Performance Tuning

## Introduction

This guide provides insights into optimizing search queries and dashboard performance in UTMStack version 10.9.0. It is part of the Performance and Optimization group, focusing on enhancing UTMStack's efficiency and scalability.

## Dashboard Management

### Creating and Managing Dashboards

UTMStack's Dashboard List offers a streamlined interface for users to search, view, manage, import, and customize dashboards. Key functionalities include:

- **Search Box**: Quickly retrieve dashboards by typing the name or associated keywords.
- **Dashboard Listings**: View all available dashboards with details such as name, description, last modified date, and direct URL access. Users can reorder the list by name or modification date and perform actions like copy, edit, and delete.

For more details, visit the [Dashboards List documentation](https://documentation.utmstack.com/v10/dashboards/dashboard_list).

### Exporting Dashboards

After configuring your dashboard, you can save it with a name and optional description. The dashboard can be exported to a PDF file for offline use. For more information, refer to the [Dashboard documentation](https://documentation.utmstack.com/v10/dashboards/dashboard).

## Index Management

### Managing Indices

Index management in UTMStack is crucial for maintaining performance. Users can delete indices, monitor their size and status, and filter them via search. Regular monitoring of indices is recommended to ensure optimal performance.

For detailed instructions, see the [Index Management documentation](https://documentation.utmstack.com/v10/configuration/index_management).

### Data Retention

UTMStack manages data retention through index rollover, snapshot archiving, and hot/cold storage configurations. These features help optimize log storage and maintain system performance.

- **Index Rollover**: Automatically deletes indices based on data size or age.
- **Snapshot Archiving**: Allows for efficient data storage and retrieval.

Learn more about data retention in the [Data Retention documentation](https://documentation.utmstack.com/v10/configuration/data_retention).

## Visualization and Query Optimization

### Configuring Visualizations

The UTMStack interface allows users to select index patterns, run chart previews, and customize chart data and appearance. Key functionalities include:

- **Metric Aggregation**: Options like count, average, sum, and min to analyze data.
- **Bucket Aggregation**: Essential for interactive features and data point redirection.

For more on creating visualizations, refer to the [Visualization documentation](https://documentation.utmstack.com/v10/dashboards/creatingvisualization).

## Summary

This guide provides an overview of optimizing search queries and dashboard performance in UTMStack. By effectively managing dashboards, indices, and visualizations, users can enhance system performance and scalability. For more detailed information, refer to the linked documentation pages.
