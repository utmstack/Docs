## Data Ingestion and Management

UTMStack utilizes various schemas and configurations to manage data ingestion effectively. Key components include:

- **Input Configurations:** Define how data is ingested, including properties like `inputPrettyName`, `inputPlugin`, and `inputWithSsl`.
- **Logstash Pipelines:** Use `UtmLogstashPipelineDTO` to process and route log data.
- **Filters:** `UtmLogstashFilter` objects allow for customization and management of data filtering processes.
- **Data Columns:** Define the structure of data with properties such as `label`, `field`, and `type`.

## Log Analysis and Query Management

The platform supports robust log analysis capabilities through:

- **Log Analyzer Queries:** Manage queries with properties like `name`, `description`, `owner`, and `creationDate`.
- **Policy Settings:** Configure data retention and snapshot policies using `PolicySettings`.
- **Audit Events:** Retrieve and manage audit events through defined API endpoints.

## Visualization and Reporting

UTMStack provides multiple visualization options to represent data effectively:

- **Chart Types:** Includes `TAG_CLOUD_CHART`, `HORIZONTAL_BAR_CHART`, `VERTICAL_BAR_CHART`, `LINE_CHART`, `AREA_CHART`, `HEATMAP_CHART`, and more.
- **Table and Chart Structures:** Use `TableType`, `BarType`, and `PieType` to structure data for visual representation.

## Summary

UTMStack supports a diverse array of data sources and formats, enabling comprehensive log management and analysis. With robust schemas and configurations, the platform facilitates efficient data ingestion, processing, and visualization. This documentation serves as a reference for configuring and managing these data sources within UTMStack.
